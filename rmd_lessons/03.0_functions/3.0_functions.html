If you ever forget what a function does you can enter the function name after ? and your computer will tell you! This will pull up a description of the function and information on how to use it.

Hint: Type the question mark followed by the function name: ?round

From the ?round help description, it specified the argument x as the input, and digits as the number of decimal places.


Using the round function with complete argument labels. Enter x=3.14 and digits=0 as the arguments into the function round().

Hint: Use round() with both argument labels: x=3.14 and digits=0 separated by a comma.

Let’s try switching the order of the arguments. Copy your code from the last command and switch the position of the arguments (digits=0, x=3.14).

Hint: Put digits=0 first, then x=3.14, separated by a comma inside round().

Switching the order doesn’t change when you use argument labels (ex: x= and digits =).


Now let’s try removing the argument labels (x= and digits =) from your last line of code. Type round(0, 3.14).

Hint: Type round(0, 3.14) - just the numbers without the x= and digits= labels.

When you have argument labels R can understand what you are asking, but without the argument labels it will assume the default order or arguments.


So argument order doesn’t matter as much when you use labels, but if you don’t use labels, you have to be mindful of argument order (the order the arguments are entered).
